Update the project state document after completing work.

1. Read the current state: `.planning/STATE.md`
2. Read the current `package.json` for version info
3. Ask the user what was completed if not clear from context

4. Update `.planning/STATE.md` with:
   - Move completed items from "Next Steps" to "Done" (or equivalent section)
   - Add new items to "Next Steps" if identified during implementation
   - Update the current version if it changed
   - Update any status indicators

5. If a task file was being worked on (e.g., `.planning/FEATURE_NAME.md`), mark completed phases.

6. Show a summary of changes made to STATE.md.
